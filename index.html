JavaScript is an interpreted language. A program called Javascript engine go by different names in different browsers but they are just code names for the javascript engine of that browser [example - Firefox has Spider-Monkey, Chrome has the v8 engine, safari has Nitro etc]. It's main job is to take the JavaScript syntex that's in human readable form and convert it into MACHINE CODE so that our computer can understand our instruction.JavaScript normally makes your page come to life and user friendly.

The JavaScript engine has a list of events that need to be processed. The event loop handles those events one at a time until the list is empty. JavaScript executes all operations on a single thread. JavaScript has a runtime model based on an event loop, which is responsible for executing the code, collecting and processing events, and executing remaining sub-tasks. It runs the asynchronous code and adds the promise callback (or any code after the await) to the event list, if there is any. Then, after anything else that was in the event list is called, it will call the callback (or the rest of the code after the await), but that code will have to wait in line until the event loop gets to it.

The main difference between local-Storage and session-Storage is the expiration date.
local-Storage

local-Storage can hold the storage across browser sessions. So even if the domain is opened in a new tab, or the browser is closed and reopened, the storage is saved. There is no expiration.

session-Storage

session-Storage is cleared when the page session ends. A page session lasts as long as the tab or the browser is open. Also, when you open a new tab or a new window, this creates a new session and new storage.


But local-Storage and session-Storage are almost the same. Both have the following rules:
  • They store key and value pairs for a particular domain.
  • The values stored may only be strings.
  • They have a limit of around 5MB of storage space.
